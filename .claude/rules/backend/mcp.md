# MCP Adapter Rules

`eventum/mcp/` is a transport-only driver adapter (peer to `api/`,
`cli/`, `server/`). It exposes Eventum to MCP clients; it holds no
business logic.

## Boundaries

- **No business logic.** Tools/resources/prompts are thin handlers that
  delegate to `core/`, `app/`, and the plugin machinery - the same
  services `api/` uses.
- **Never import `eventum.api`.** Share via `core/`+`app/`, never call
  the HTTP API over the network, never reach into `api/routers/*`.
- **Capability via DI, not branching.** Tools depend on a context
  Protocol (`AuthoringContext`, later `LiveContext`); they must not
  branch on transport or concrete context type. Live-only tools are
  registered only by the root that supplies a `LiveContext`.

## Registration

- Each tool/resource/prompt module exposes `register(mcp, context, ...)`
  and is wired in `build_server`. Tools take `*, transport`; resources
  and prompts take only what they use.
- The handler is a nested closure over `context` (so `context` never
  appears in the agent-facing schema).

## Errors and secrets

- Catch `ContextualError` subclasses and return `to_tool_error(...)` -
  never raise across the boundary, never blanket-forward `.context`.
- `to_tool_error` allow-lists keys, relativises absolute paths against
  `generators_dir`, and redacts provided secret values. Tools that load
  configs (validate/preview) pass `redact_values` (the resolved
  secrets) so none leak into error text. No absolute path and no secret
  value may reach the agent.

## Concurrency (live tools, Plan 3)

- Every `GeneratorManager`/`Generator` call runs in `asyncio.to_thread`
  (mirrors `api/routers/generators/routes.py`); these calls block and
  must not run on the event loop.

## Logging

- stdio: stdout is the JSON-RPC channel - all logs go to stderr/file
  (`logconf.use_stderr`). No log handler may target stdout.
- Forwarded tool failures are logged via
  `observability.observe_failure` with `mcp_tool` + `mcp_transport`.
  New log fields go in `LOGGING.md` first.

## Templating reference

- `eventum://templating/reference` is generated by introspection
  (`plugins/event/plugins/template/reference.py`); do not hand-maintain
  the helper list.
